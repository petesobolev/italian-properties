Italian Properties - Architecture Diagram

Next.js 16 Full-Stack Real Estate Platform

Frontend (Client)
Public Pages
Next.js App Router + React
Home, Region listings, Property details, Archived listings
Admin Portal
React Client Components
Token-based agent dashboards for property management
Interactive Components
React + Leaflet
Image gallery, Map views, Contact form, Filters
Backend (Server)
Server Components
Next.js RSC
Server-rendered pages with direct DB access
API Routes
Next.js Route Handlers
/api/contact, /api/admin/*, /api/search
Data Access Layer
TypeScript + pg
lib/properties.ts, lib/admin.ts, db/connection.ts
External Services
Supabase PostgreSQL
Database + RLS
Properties, Sources, Regions tables with Row Level Security
Vercel Blob
Object Storage
Property images, videos, agent logos
Resend
Email API
Property inquiry emails to agents
Data Ingestion (Scrapers)
Scraper Framework
TypeScript + node-html-parser
Common ingestion pipeline with upsert logic
Source Scrapers
Per-agency adapters
Vittori, Casa Amola (extensible)
Translation Service
Google Translate API
Italian → English for descriptions
Enrichment Scripts
Claude AI + Geocoding
Feature extraction, address geocoding
Hosting & Deployment
Vercel
Edge + Serverless
Auto-deploy from Git, environment variables, CDN
Domain
italianproperties.co
Custom domain with SSL

Data Flow

Browser → Next.js Server → PostgreSQL Page requests fetch data via Server Components
Contact Form → /api/contact → Resend API → Agent Email
Admin Upload → /api/admin/upload → Vercel Blob Images stored, URL saved to DB
Scraper → Source Website → Translate → PostgreSQL
Admin Portal → Token Validation → CRUD Operations Agents manage their own listings
